# Contributing

Contributions to the Decibri documentation are welcome, including from integration partners adding or improving provider pages. The site is small, every change goes through pull request review, and structural decisions are made collaboratively.

## Partner PR guidance is coming soon

Detailed partner contribution guidance, including provider page templates and a step-by-step walkthrough, will land in a later commit. Until then, if you are planning a substantial change such as a new integration provider page, structural reorganisation, or a new documentation section, please open an issue first so we can coordinate before you spend time on the work. Small fixes (typos, broken links, factual corrections, prose clarifications) do not need an issue first; a pull request is fine.

## Conventions

- Use `Decibri` capitalised in user-facing prose, not `decibri`.
- No em dashes anywhere in site content. Rewrite using periods, commas, colons, or parentheses.
- No commercial claims. Avoid words like "best", "fastest", "industry-leading", or comparative claims against other libraries. Documentation describes what Decibri does, not how good it is.
- API code examples must match the actual Decibri API. The source of truth is the Decibri code repository at [https://github.com/decibri/decibri](https://github.com/decibri/decibri). Do not write API identifiers, signatures, or error message strings from memory.
